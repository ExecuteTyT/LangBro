# LangBro — AI English Challenge Bot
## Техническое задание v1.0

**Дата:** 01.04.2026
**Продукт:** Telegram-бот для группового челленджа по изучению английского языка
**Контекст:** Бот встраивается в активный челлендж (~20 участников, ежедневные отчёты, цель — B2)
**LLM Provider:** Google AI Studio — Gemini 2.5 Flash
**Разработка:** Claude Code

---

## 1. Обзор продукта

### 1.1 Проблема
Группа из 20 человек ведёт челлендж по изучению английского. Каждый день пн-пт участники скидывают в чат отчёты о том, что сделали. Проблемы:
- Отчёты тонут в потоке сообщений
- Нет наглядной статистики прогресса
- Нет механизма мотивации кроме самодисциплины
- Сложно отследить кто отчитался, а кто слился
- Нет персонализированной практики

### 1.2 Решение
AI-бот «LangBro» — полноценный участник группы, который:
- Принимает и парсит отчёты в свободной форме через LLM
- Ведёт статистику по каждому участнику
- Генерирует ежедневные сводки
- Проводит квизы и даёт задания
- Мотивирует через геймификацию (streaks, баттлы, лидерборды)
- Работает как AI-тренер в личке

### 1.3 Персона бота
**Характер:** Дружелюбный тренер-бро. Подкалывает, хвалит, вызывает на батл.
**Язык:** Русский с подмешиванием английского. Процент английского зависит от уровня участника:
- Beginner (A1-A2): 10% английского
- Intermediate (B1): 30% английского
- Upper-Intermediate (B1+): 50% английского

**Тон:** Неформальный, поддерживающий, с юмором. Никогда не токсичный.

## 2. Функциональные требования

### 2.1 Регистрация и онбординг

#### FR-001: Добавление бота в группу
- Админ добавляет бота в существующий групповой чат
- Бот отправляет приветственное сообщение с описанием функционала
- Бот НЕ создаёт новый челлендж автоматически — ждёт команду от админа

#### FR-002: Создание/настройка челленджа
- Команда: `/create_challenge`
- Только для админов группы
- Параметры через интерактивное меню (InlineKeyboard):
  - Название челленджа (текст)
  - Расписание (пн-пт / ежедневно / кастом)
  - Время дедлайна отчёта (по умолчанию 23:59 MSK)
  - Время Daily Digest (по умолчанию 22:00 MSK)
  - Часовой пояс группы (по умолчанию MSK)

#### FR-003: Приглашение участников (Launch)
- Команда: `/launch` в группе (только админ)
- Бот отправляет в группу красивое сообщение-приглашение с InlineKeyboard-кнопкой «🚀 Присоединиться»
- Кнопка — deep link: `t.me/{bot_username}?start=join_{challenge_id}`
- Админ может вызвать `/launch` повторно (для опоздавших)
- Бот НЕ тегает участников — каждый нажимает кнопку сам

#### FR-003a: Онбординг участника (через deep link)
- Участник нажимает кнопку → открывается личка с ботом
- Автоматически срабатывает `/start join_{challenge_id}`
- Бот распознаёт challenge_id и запускает анкету в личке:
  1. Имя — как обращаться (текстовый ввод)
  2. Текущий уровень английского (A1 / A2 / B1 / B2 — InlineKeyboard)
  3. Основная цель (разговорный / рабочий / экзамен / общее развитие — InlineKeyboard)
- После заполнения:
  - Участник добавлен в челлендж (запись в `user_challenges`)
  - Бот пишет в ГРУППУ: «Айрат в деле! 🤝» (через chat_id из challenges)
  - Бот пишет в ЛИЧКУ: краткая инструкция (как отправлять отчёт, основные команды)
- Если участник уже в этом челлендже — бот говорит: «Ты уже участвуешь! 💪»
- Если бот не может написать в группу (был удалён) — сообщает участнику в личке

#### FR-003b: Присоединение к нескольким челленджам
- Один пользователь может участвовать в нескольких челленджах одновременно
- Каждый deep link привязывает к конкретному челленджу через challenge_id
- Анкета (уровень, цель) заполняется ОДИН РАЗ при первом взаимодействии с ботом
- При присоединении к новому челленджу бот использует уже заполненные данные
- Участник может обновить уровень/цель через `/profile` в личке

#### FR-004: Просмотр участников
- Команда: `/members`
- Показывает список участников с текущим streak и уровнем

#### FR-005: Переключение активного челленджа (для личных сообщений)
- При отправке команды в ГРУППУ — бот автоматически привязывает к челленджу этой группы
- При отправке команды в ЛИЧКУ бота:
  - Если у пользователя 1 активный челлендж → автопривязка, без вопросов
  - Если у пользователя 2+ активных челленджа → InlineKeyboard: «В каком челлендже?» + кнопки с названиями
  - Выбранный челлендж сохраняется как `active_challenge_id` в таблице `users`
  - Активный контекст сохраняется до явного переключения
- Команда `/switch` — принудительно переключить активный челлендж в личке
- В каждом ответе в личке бот показывает текущий контекст: `[📌 Английский до B2]`

### 2.2 Приём и парсинг отчётов

#### FR-010: Приём отчёта в группе
- Участник пишет отчёт в свободной форме в группу
- Триггер: команда `/report` перед текстом ИЛИ ответ (reply) на сообщение бота с напоминанием
- Бот обрабатывает текст через Gemini 2.5 Flash

#### FR-011: Приём отчёта в личке
- Участник пишет `/report` в личку боту
- Маршрутизация по контексту:
  - 1 активный челлендж → автопривязка
  - 2+ челленджей → используется `active_challenge_id` из профиля
  - Если active_challenge_id не задан → InlineKeyboard с выбором
- Бот парсит и добавляет в статистику нужного челленджа
- В ответе бот показывает контекст: `[📌 Английский до B2]`

#### FR-012: LLM-парсинг отчёта
Бот отправляет текст отчёта в Gemini с промптом для структурирования.

**Входные данные:** свободный текст отчёта
**Выходной формат (JSON):**
```json
{
  "activities": [
    {
      "category": "speaking",
      "description": "Рабочий созвон на английском",
      "duration_minutes": 40,
      "details": null
    },
    {
      "category": "vocabulary",
      "description": "Учил новые слова",
      "count": 15,
      "details": null
    },
    {
      "category": "listening",
      "description": "Подкаст без субтитров",
      "duration_minutes": 30,
      "details": "TED Talk"
    }
  ],
  "total_points": 75,
  "summary": "Продуктивный день: созвон + подкаст + слова"
}
```

**Категории активностей:**
| Категория | Код | Баллы |
|-----------|-----|-------|
| Разговорная практика | `speaking` | 2 балла/мин |
| Аудирование (подкасты, сериалы) | `listening` | 1 балл/мин |
| Чтение | `reading` | 1.5 балла/мин |
| Письмо | `writing` | 2 балла/мин |
| Словарный запас | `vocabulary` | 3 балла/слово |
| Грамматика | `grammar` | 5 баллов/урок |
| Приложения (Duolingo и т.д.) | `app_practice` | 10 баллов/урок |
| Другое | `other` | 5 баллов/активность |

#### FR-013: Подтверждение отчёта
После парсинга бот отвечает участнику с:
- Разбором активностей (иконки + категории + время/количество)
- Начисленные баллы
- Обновлённый streak
- Мотивационный комментарий от бота (генерируется LLM с учётом характера бота)

**Пример ответа:**
```
✅ Отчёт принят, Айрат!

🗣 Speaking: созвон 40 мин (+80 pts)
👂 Listening: подкаст 30 мин (+30 pts)
📚 Vocabulary: 15 новых слов (+45 pts)

💰 Итого за день: 155 pts
🔥 Streak: 12 дней подряд

Nice one, bro! Созвон 40 минут — это серьёзно.
Keep that energy going 💪
```

#### FR-014: Редактирование отчёта
- Участник может отправить `/report` повторно в тот же день
- Новый отчёт ЗАМЕНЯЕТ предыдущий (не суммируется)
- Бот предупреждает: «Обновил твой отчёт за сегодня. Предыдущий перезаписан.»

### 2.3 Daily Digest (Ежедневная сводка)

#### FR-020: Автоматическая сводка
- Время: настраиваемое (по умолчанию 22:00 по часовому поясу группы)
- Отправляется в групповой чат

**Содержимое сводки:**
```
📊 Daily Digest — 31 марта, понедельник

👥 Отчитались: 16/20

🏆 Top-3 дня:
1. 🥇 Айрат — 155 pts (speaking 40мин + подкаст + 15 слов)
2. 🥈 Тимур — 120 pts (Duolingo + статья + speaking)
3. 🥉 Марат — 85 pts (сериал 2 эпизода + грамматика)

🔥 Streak-лидеры:
   Айрат: 23 дня | Рустам: 18 дней | Лена: 15 дней

📈 Камбэк дня: Руслан (вернулся после 3 дней!)

😴 Не отчитались (ещё есть время до 23:59):
   @user1 @user2 @user3 @user4

📊 Группа за сегодня:
   ⏱ 14ч 20мин суммарной практики
   📚 87 новых слов на всю группу
   🗣 3ч 40мин разговорной практики

💬 Fun fact: Сегодня группа наговорила на английском
   больше, чем длится фильм "Властелин Колец" (расширенная версия) 🎬
```

#### FR-021: Напоминание об отчёте
- Время: 20:00 (за 4 часа до дедлайна, настраиваемо)
- Отправляется в группу
- Тегает ТОЛЬКО тех, кто ещё не отчитался
- Текст с юмором (генерируется LLM, варьируется каждый день)

#### FR-022: Еженедельная сводка
- День: воскресенье в 12:00
- Расширенная статистика за неделю:
  - MVP недели
  - Рейтинг всех участников
  - Рост/падение позиций
  - Лучший streak
  - «Камбэк недели» (самый большой рост после провала)
  - «Рокет» (самый большой прирост баллов)
  - Общая статистика группы (часы, слова, и т.д.)
  - Результаты Weekly Battles (см. FR-042)

### 2.4 Word of the Day (Слово дня)

#### FR-030: Утренний пост
- Время: 8:00 (настраиваемо)
- Только в рабочие дни челленджа (пн-пт по умолчанию)
- Генерируется через LLM

**Содержимое:**
```
🌅 Good morning, bros!

📖 Word of the Day: "nevertheless"
🔊 /nevəðəˈles/
📝 Тем не менее, всё же, несмотря на это

💡 Примеры:
• "The project was difficult. Nevertheless, we finished on time."
• "I was tired. Nevertheless, I went to the gym."

🏷 Уровень: B1
🎯 Связано с тем, что учили вчера: "however", "although"

📌 Challenge: Используй "nevertheless" сегодня в речи или письме
   и расскажи в отчёте, в каком контексте использовал!
```

#### FR-031: Подбор слова
- LLM подбирает слово, учитывая:
  - Средний уровень группы
  - Слова, которые уже были (не повторяет)
  - Тематические связи с предыдущими словами
  - Практическую полезность

#### FR-032: Трекинг использования
- Если участник в отчёте упоминает Word of the Day — бонусные баллы (+20 pts)
- LLM при парсинге отчёта проверяет это

### 2.5 Pop Quiz (Дневной квиз)

#### FR-040: Квиз в группу
- Время: рандомное в интервале 12:00–15:00 (чтобы был элемент неожиданности)
- Только в рабочие дни
- Формат: один вопрос с вариантами ответов (InlineKeyboard, 4 кнопки)

**Типы вопросов (рандомно):**
1. **Перевод слова/фразы** — из слов, которые группа учила
2. **Выбери правильное время** — грамматика в контексте
3. **Заполни пропуск** — идиома или устойчивое выражение
4. **Что означает фразовый глагол** — give up, look into, и т.д.
5. **Найди ошибку** — предложение с одной ошибкой
6. **Синоним/антоним** — подбери ближайшее по значению

#### FR-041: Механика ответов
- Каждый участник может ответить только один раз (кнопка блокируется)
- Результат виден сразу (бот отвечает правильно/неправильно в личку, чтобы не спойлерить)
- Через 30 минут (или когда все ответили) бот публикует результаты:
  - Правильный ответ с объяснением
  - Кто ответил правильно
  - Кто ответил первым (бонус +10 pts за скорость)
  - Все правильные ответы: +15 pts

#### FR-042: Weekly Battle
- Каждый понедельник бот рандомно создаёт пары из участников
- Нечётное количество — один участник получает «bye» (автоматически проходит)
- В пятницу бот сравнивает суммарные баллы пар за неделю
- Проигравший получает «штраф-задание» (генерируется LLM):
  - Записать голосовое на английском на заданную тему (30 сек)
  - Написать короткий пост на английском
  - Перевести мем на английский
- Результаты батлов публикуются в пятничной сводке

### 2.7 AI Coach (Личка)

#### FR-060: Разбор текста на ошибки
- Участник пишет в личку: `/check <текст на английском>`
- Бот разбирает грамматику, стиль, предлагает варианты
- Объясняет ошибки на русском, на уровне, понятном участнику

#### FR-061: Ролевой диалог
- Команда: `/practice`
- Бот предлагает сценарии (InlineKeyboard):
  - Job interview
  - Ordering at a restaurant
  - Small talk at a party
  - Business meeting
  - Кастомный сценарий
- Бот ведёт диалог на английском, подстраиваясь под уровень
- После окончания — feedback с разбором ошибок

#### FR-062: Объяснение слова/фразы
- Команда: `/explain <слово или фраза>`
- Объяснение с примерами, синонимами, контекстом использования
- Адаптировано под уровень участника

#### FR-063: Перевод с разбором
- Команда: `/translate <текст>`
- Перевод + объяснение грамматических конструкций
- Идиомы и устойчивые выражения подсвечиваются отдельно

### 2.8 Администрирование

#### FR-070: Управление челленджем
- `/settings` — настройки (только админ):
  - Изменить расписание
  - Изменить время сводки/напоминаний
  - Вкл/выкл отдельные фичи (квизы, WotD, батлы)
  - Управление баллами (множители)

#### FR-071: Удаление участника
- `/kick @username` — убрать из челленджа (только админ)
- Статистика участника архивируется, но удаляется из активного рейтинга

#### FR-072: Пауза челленджа
- `/pause` — поставить челлендж на паузу (выходные, праздники)
- Streak не сбрасывается во время паузы
- Бот не отправляет напоминания и сводки

#### FR-073: Пауза участника
- `/vacation <количество дней>` — личная пауза
- Streak замораживается
- Максимум 7 дней подряд, 14 дней в месяц
- Бот отмечает в сводке: «Айрат на каникулах (осталось 3 дня)»

### 2.9 Voice Messages (Голосовые сообщения)

Бот умеет говорить на английском — отправляет короткие голосовые сообщения для тренировки произношения. Реализация через Edge TTS (бесплатно, нейросетевые голоса Microsoft, без API-ключа).

**Технический пайплайн:** Текст → Edge TTS → MP3 → ffmpeg конвертация → OGG Opus → Telegram voice message.

#### FR-080: Голосовое в Word of the Day
- После текстового сообщения WotD — бот отправляет голосовое (5-8 сек):
  - Произношение слова (медленно, чётко)
  - Пауза 1 сек
  - Первый пример из контекста (нормальная скорость)
- Голос: `en-US-GuyNeural` (мужской, нейтральный американский) или `en-GB-RyanNeural` (британский) — настраивается
- Скорость речи для слова: `-20%` (замедленная), для примера: `+0%` (нормальная)

#### FR-081: Listening Quiz (новый тип квиза)
- Бот отправляет ТОЛЬКО голосовое (без текста) с фразой/предложением на английском
- Участники должны понять, что сказано, и выбрать правильный вариант:
  - «Что было сказано?» — 4 варианта текстом
  - Или: «Что означает эта фраза?» — 4 варианта перевода
- Можно переслушать (голосовое остаётся в чате)
- Бонусные баллы: +20 pts (сложнее текстового квиза)

#### FR-082: Голосовое в ответе на отчёт
- Короткая мотивационная фраза на английском (2-4 сек)
- Отправляется НЕ каждый раз, а по условиям (чтобы не надоедать):
  - Streak кратен 7 (7, 14, 21...): "Seven days in a row, that's impressive!"
  - Новый рекорд по баллам: "New personal record, keep pushing!"
  - Камбэк после пропуска: "Welcome back, we missed you!"
  - Рандомно 1 раз из 5 отчётов: короткая фраза-похвала
- Фразы предгенерируются и кешируются (не генерировать TTS на каждый отчёт)

#### FR-083: Команда `/pronounce <слово или фраза>`
- Работает в группе и в личке
- Бот отправляет голосовое с произношением
- Для одного слова: медленно → пауза → нормальная скорость
- Для фразы: нормальная скорость
- Можно указать акцент: `/pronounce -uk nevertheless` (британский) или `/pronounce -us nevertheless` (американский)
- По умолчанию — акцент, выбранный в настройках челленджа

#### FR-084: Голос в AI Coach
- `/check` — после разбора ошибок бот произносит исправленную версию
- `/practice` — бот может отвечать голосовыми в ролевом диалоге (участник тоже может слать голосовые → Speech-to-Text через Whisper API для распознавания)
- `/explain` — произношение объясняемого слова + примеры

#### FR-085: Кеширование голосовых
- Часто используемые фразы (мотивационные, WotD) кешируются как файлы
- Ключ кеша: hash(text + voice + rate)
- Хранение: локальная файловая система, папка `voice_cache/`
- TTL: 30 дней, автоочистка по cron
- Экономит время генерации при повторном использовании

#### FR-086: Настройки голоса (админ)
- `/settings` → Voice Settings:
  - Вкл/выкл голосовые сообщения
  - Выбор акцента: American 🇺🇸 / British 🇬🇧 / Australian 🇦🇺
  - Выбор голоса: мужской / женский
  - Частота голосовых в ответах: always / sometimes / never

## 3. Нефункциональные требования

### 3.1 Производительность
- Ответ на команду: < 3 секунды
- Парсинг отчёта (LLM): < 10 секунд
- Daily Digest генерация: < 15 секунд
- Одновременно обслуживать минимум 5 групп / 100 участников

### 3.2 Надёжность
- Бот должен работать 24/7
- При падении — автоматический рестарт (systemd или docker)
- Логирование всех LLM-запросов и ответов для дебага
- Retry-логика для LLM API (3 попытки с экспоненциальным backoff)

### 3.3 Стоимость
- Gemini 2.5 Flash — бюджетная модель для основных операций
- Кеширование повторяющихся промптов (system prompts)
- Оценочный расход: ~$5-15/месяц на группу из 20 человек

### 3.4 Безопасность
- API ключи хранятся в .env, не коммитятся
- Данные пользователей не передаются третьим лицам
- Текст отчётов хранится в БД для аналитики, но может быть удалён по запросу

### 3.5 Часовые пояса
- Все времена хранятся в UTC
- Конвертация при отображении по часовому поясу группы (настраивается)
- По умолчанию: Europe/Moscow (MSK, UTC+3)

## 4. Интеграции

### 4.1 Telegram Bot API
- Библиотека: `aiogram 3.x`
- Webhook или long polling (для MVP — long polling)
- Inline keyboards для квизов и меню
- Callback queries для обработки ответов

### 4.2 Google AI Studio (Gemini 2.5 Flash)
- API: `google-generativeai` Python SDK
- Модель: `gemini-2.5-flash`
- Structured output (JSON mode) для парсинга отчётов
- System instructions для персоны бота

### 4.3 Edge TTS (Text-to-Speech)
- Библиотека: `edge-tts` (Python, pip install)
- Бесплатно, без API-ключа, нейросетевые голоса Microsoft
- Голоса для английского:
  - `en-US-GuyNeural` — мужской, американский (рекомендуемый по умолчанию)
  - `en-US-AriaNeural` — женский, американский
  - `en-GB-RyanNeural` — мужской, британский
  - `en-GB-SoniaNeural` — женский, британский
  - `en-AU-WilliamNeural` — мужской, австралийский
- Выход: MP3 → конвертация в OGG Opus через `ffmpeg`
- Параметры: rate (скорость), pitch (высота), volume
- Кеширование сгенерированных файлов для экономии

### 4.4 ffmpeg (аудио конвертация)
- Конвертация MP3 → OGG Opus для Telegram voice messages
- Команда: `ffmpeg -i input.mp3 -c:a libopus -b:a 64k output.ogg`
- Должен быть установлен на сервере (`apt install ffmpeg`)

### 4.5 Scheduler
- `APScheduler` для cron-задач:
  - 08:00 — Word of the Day
  - 12:00-15:00 — Pop Quiz (рандомное время)
  - 20:00 — Напоминание
  - 22:00 — Daily Digest
  - Понедельник 09:00 — Weekly Battles (создание пар)
  - Воскресенье 12:00 — Weekly Digest
